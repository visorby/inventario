Caso de Uso

Definición

Un Caso de Uso es una secuencia de acciones que describe una interacción entre un actor y el sistema, mostrando cómo el sistema responde a una solicitud del usuario.

Casos de Uso del Sistema:
  • UC001: Registrar Entrada de Producto

    El encargado registra un nuevo producto en el almacén.

  • UC002: Consultar Disponibilidad de Stock

    El usuario consulta la disponibilidad de un producto específico.

  • UC003: Generar Salida de Producto

    Se registra la salida de un producto del almacén.

  • UC004: Generar Reporte de Inventario

    Sistema genera un reporte del estado actual del inventario.

Caso de Uso de Negocio (CUN)

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

Definición

Un Caso de Uso de Negocio describe procesos de negocio de mayor nivel que pueden involucrar múltiples casos de uso del sistema.

CUNs del Sistema:

Proceso completo que incluye la verificación, registro e ingreso de nuevos materiales al almacén. Involucra múltiples casos de uso técnicos.

Proceso de entrega de materiales a proyectos o clientes finales, incluyendo preparación, empaque y documentación.

Proceso de monitoreo, auditoría y reporte del estado del inventario en tiempo real.

Caso de Uso Expandido

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Los casos de uso expandidos proporcionan un nivel de detalle completo del flujo de acciones.

Definición

Un Caso de Uso Expandido describe en detalle cada paso de una interacción, incluyendo el flujo principal, flujos alternativos, condiciones previas y postcondiciones.

Ejemplo: UC001 - Registrar Entrada de Producto
Nombre Registrar Entrada de Producto
Actor Primario Encargado de Almacén
Precondición El encargado ha autenticado en el sistema
Flujo Principal 1. El encargado accede al módulo de entrada
2. Ingresa los datos del producto
3. Sistema valida la información
4. Se registra la entrada en la base de datos
5. Sistema genera comprobante
Flujos Alternativos • Datos inválidos: Sistema muestra error y solicita corrección
• Producto duplicado: Sistema advierte sobre existencia previa
Postcondición El producto está registrado y disponible en el inventario
